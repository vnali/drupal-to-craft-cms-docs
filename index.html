# Introduction

This plugin is a Craft CMS (opens new window) plugin -currently available as a service- for importing content from Drupal to Craft CMS based on Craft's Feed Me (opens new window) plugin.

# Requirement

  • Drupal higher than 8.7.x.

    • HTTP Basic Authentication and JSON:API core modules.
    • Drupal user to read JSON:API.

  • Latest version of Craft 3

TIP

Drupal 6 and 7 can be converted to 8 or 9 by Drupal core Migrate modules (opens new window).

# How migration works

This plugin connects to Drupal via JSON API.
At first, the plugin fetches Drupal fields and generates a mapping table as shown in the image below.
After mapping Drupal to Craft fields, the plugin creates requested custom fields (opens new window) -it can be a simple field -Plain text, Dropdown, ...- , a column in a table field (opens new window), or even more complex a field in super table field (opens new window) or a field in a Matrix field (opens new window)- and then generates feeds based on the Feed-Me plugin.
After running feeds, data is migrated to Craft CMS.

The plugin can detect Drupal Languages and migrate each language to a related Craft site (opens new window). Drupal content with 'Not Specified' and 'Not Applicable' language can be migrated to the selected Craft site in the language mapping table.

# User

Drupal users and custom fields can be converted to Craft's users (opens new window).

# Media and file

The plugin migrates media and files to Craft assets (opens new window) in two ways:

  • After copying the Drupal files to Craft, files are added to Craft by Asset index utility (opens new window).
  • Media and files are migrated to Craft by Feed-Me based on the media type, file extension.

The plugin helps to migrate files meta -description, display, alt, title- when the migration process happens.
Currently files and media can be migrated to local volume. Embedded Assets (opens new window) is supported.

# Taxonomy

Taxonomies in Drupal can be converted to Craft's entries, categories (opens new window), and tags (opens new window).
Tags in Craft have no hierarchy, so taxonomies with parent->child relationship should be migrated to Craft entries or categories.

# Block

Blocks can be migrated to Craft CMS in 3 ways:

  • each block is converted to one Entry (opens new window). Channel or Structure Section for each block type is created manually before migration.
  • each block is converted to a Single (opens new window). all required Single Sections and their fields are created automatically.
  • each block is migrated to Global (opens new window). The Global set and its fields are created automatically.

# Node

  • Each node is migrated to entry. We create a Channel (opens new window) or Structure (opens new window) Section for each block type before migration. Fields are migrated automatically.

# Menu

This plugin can migrate menus to Structure Sections (opens new window) or Navigation (opens new window) plugin.

TIP

For all Drupal items, fields are created based on field mapping and added to Craft field layouts (opens new window) automatically.

TIP

This plugin tries to convert an internal Drupal link address to Craft internal link. for example, if there is a Drupal link field with a URL like /node/1 or /{node1-alias}, we try to get the address of that node on Craft and replace link URL with it.

# Fields

This plugin supports the migration of most Drupal fields to the Craft core field.

# Supported Drupal modules

  • Address field (opens new window)
    All parts of the Address field can be migrated to fields or columns in a table, Super Table, or matrix field in Craft CMS.
    Country code can be migrated to plain text or drop-down in Craft. When the country part of the Address field is migrated to drop-down in Craft, all country codes are automatically added to drop-down options.

  • Block field (opens new window)
    Block field is migrated to a text field where the text field's value is the UUID of that migrated block. You can query that block on Craft later.

  • Color field (opens new window)
    The plugin can convert the Hex Color and opacity of that color to the Craft, where Color value is converted to Craft Color field and opacity to a plain text or number field. Color and opacity in Craft can be paired in a Matrix, super table, or table.

  • Computed field (opens new window)
    Only computed field and their values -and not how field is computed- can be migrated to Craft field.

  • Physical field (opens new window)
    This plugin migrates all types of physical fields (area, dimension, length, measure, temp, volume, weight) to Craft. All physical units are added to Craft drop-down options.

  • Geo field (opens new window)

  • Paragraph field (opens new window)
    Paragraph fields can be converted to the table, super table, or Matrix field based on the fields used inside that Paragraph field.

  • Table field (opens new window)
    Table field can be converted to table field in Craft.

  • Video field (opens new window)
    Video field upload type can be migrated to Asset in Craft. Video field embed type can be migrated to plain text, url or Embedded Assets (opens new window).

  • Youtube field (opens new window)
    Youtube field addresses can be migrated to Craft plain text or URL or Embedded Assets (opens new window).

# FAQ

How can I download this plugin?

Currently, the plugin is only available as a service.

Is this plugin stable?

We tested this plugin with some Drupal sites, and it worked well. We need more Drupal sites to migrate and make this plugin stable.

I used a Drupal field which is not available on the supported list. What can I do?

Please let us know which Drupal field are you using. We will work to add support for it.

Is multilingual Drupal site is supported?

Yes, you can migrate multilingual Drupal sites to different Craft sites.

Is migrating sites with Domain module supported?

Currently, we are working on it, and we need more Drupal sites to test the migration process. Don't hesitate to contact us if you have a Drupal site with the Domain module.

Is migrating Drupal multi-site supported?

Currently, we are working on it, and we need more Drupal sites to test the migration process. If you have a Drupal multi-site, please get in touch with us.

What happens if I am not satisfied with the result of migration?

At first step, we try to migrate some items of your Drupal site for test and demonstartion purpose. If you are not satisfied with the result, there is no need to pay for it.
